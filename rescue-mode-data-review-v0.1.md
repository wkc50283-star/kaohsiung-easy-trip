# 備援模式第一版資料審核表 v0.1

本文件整理「一路玩高雄 / 高雄不開車自由行助手」備援模式第一版資料審核表。

本文件只供人工確認與後續資料建模參考。現階段不得直接視為已驗證功能資料，也不得直接轉成前台推薦結果。

## 1. 文件目的

備援模式用來處理旅途中出現變數時的保守改道判斷，例如走歪、睡過頭、提早玩完、天氣變差、體力下降、心情改變、只想吃美食逛街、跑到外縣市或未知地點。

第一版資料審核目標：

1. 先整理旅遊圈、辨識點、回收點與情境規則。
2. 標出哪些資料可人工判斷，哪些必須官方確認。
3. 保留 Needs review，避免把未確認資訊寫成確定答案。
4. 為未來轉成 JSON 或功能資料做準備，但本文件本身不是程式資料。

## 2. 使用原則

核心原則：

- 不要害旅人越走越遠。
- 如果使用者還想繼續玩，要提供分級選項：
  1. 順路可玩
  2. 可玩但較繞
  3. 不建議但可自行決定
- 第一版不接 TDX 即時 API。
- 第一版不做即時定位。
- 路線編號、站牌、方向、末班車、即時到站不得猜。
- 未經 TDX 或官方資料確認，一律標 Needs review。

資料使用限制：

1. 不顯示末班車。
2. 不承諾一定趕得上。
3. 不推薦未驗證店家。
4. 不把未知地點假裝成已知景點。
5. 不把跨縣市點硬接成高雄市區短線。
6. 不把「不建議」寫成禁止，只做風險提醒與分級。

## 7. Needs review 清單

以下項目在第一版文件階段必須保留 Needs review，不得直接寫成確定資料：

| 項目 | 為什麼需要審核 | 建議確認來源 |
|---|---|---|
| 路線編號 | 錯誤會直接影響旅人搭乘 | TDX、官方交通資料 |
| 站牌名稱與位置 | 站牌更名、位置異動會導致誤導 | TDX、地方交通官方資料 |
| 路線方向 | 上下行方向不可猜 | TDX、官方路線圖 |
| 末班車 | 不能承諾趕得上 | TDX、官方業者公告 |
| 即時到站 | 第一版不接即時 API | TDX 即時資料，未來版本才評估 |
| 渡輪班次與停航 | 天候影響大 | 官方航運公告 |
| 跨縣市客運 | 班距與末班風險高 | TDX、客運業者、地方交通官方資料 |
| 台鐵／高鐵班次 | 回程與跨縣市風險高 | 台鐵、高鐵官方資料 |
| 天氣警報 | 雨天、風大、颱風不可只靠一般描述 | 中央氣象署官方公告 |
| 無障礙與步行距離 | 長輩、小孩、行李風險 | 官方站體資訊、現場人工確認 |
| 店家與餐飲 | 不推薦未驗證店家 | 不列單一店家，僅保留類型 |
| 景點營業狀態 | 展館、活動、橋梁開放可能異動 | 官方網站或現場公告 |
