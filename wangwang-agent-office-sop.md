# wangwang-agent-office-sop.md

本文件記錄王王團隊在本專案中的工作 SOP。

此文件用於協助 Codex / AI Agent 理解角色分工、任務流、審核節點與指令撰寫方式。
若本文件與 `AGENTS.md`、`do-not-change.md` 衝突，優先遵守 `AGENTS.md` 與 `do-not-change.md`。

## 1. 團隊角色

1. 王國政是董事長，重大事項最終批准者。
2. 王王是 CEO／毒舌審核長／稽核長，負責決策、分派、否決錯誤方向、稽核成員是否走偏或偷懶。
3. 蜜醬是總控 PM，負責任務拆解與整合。
4. 花花負責前端 UI。
5. 地瓜負責後端、API、資料流。
6. 阿梟負責 QA、DevOps、PR、測試。
7. 咕嚕負責研究、資料來源、競品。
8. 啾皮負責文案、社群、內容。
9. 彩彩負責行銷企劃、影片腳本、廣告策略。
10. 豆芽負責數據營運、GA4、報表。

## 2. 標準工作流程

王王團隊標準流程如下：

1. 董事長下任務。
2. 王王判斷任務方向、範圍與風險。
3. 蜜醬拆解任務並整合工作包。
4. 相關成員處理各自負責範圍。
5. 王王稽核成果是否走偏、偷懶、過度擴張或違反限制。
6. 回報成果、檢查結果與風險。
7. 重大事項交由董事長批准。
8. 取得批准後才執行重大操作。
9. 執行後測試與驗證。
10. 留下紀錄，方便後續追蹤。

## 3. 重大事項提報規則

以下事項必須提報董事長，未經明確批准不得執行：

1. 合併 PR。
2. 修改正式網站核心功能。
3. 修改首頁主流程。
4. 修改已驗證功能。
5. 刪除資料。
6. 修改 API 或資料結構。
7. 新增金流、帳號、權限。
8. 對外發布文章、社群貼文或正式公告。
9. 任何可能造成不可逆影響的操作。

## 4. 小任務派工原則

小任務不必每次叫全部成員。

應依任務性質只叫相關角色：

1. 前端畫面、RWD、UI：花花。
2. 後端、API、資料流：地瓜。
3. 測試、PR、部署、檢查：阿梟。
4. 資料來源、研究、競品：咕嚕。
5. 文案、內容、社群語氣：啾皮。
6. 行銷活動、影片腳本、廣告策略：彩彩。
7. GA4、報表、營運數據：豆芽。
8. 任務拆解、整合、跨角色協調：蜜醬。
9. 範圍判斷、否決錯誤方向、最終稽核：王王。

若任務很小，可由王王直接判斷，不需要完整派工。

## 5. Codex 指令最省指令法

所有 Codex 指令採最省指令法。

指令應該：

1. 短。
2. 準。
3. 只寫目標。
4. 只寫必要限制。
5. 只寫紅線。
6. 只寫回報格式。

指令不應該：

1. 寫大量背景廢話。
2. 同時塞入多個不相關任務。
3. 讓 Codex 有機會自行擴大範圍。
4. 使用模糊語句，例如「順便」、「看情況」、「幫我優化一下」。
5. 讓 Codex 猜測是否可以修改未指定檔案。

## 6. 王王稽核重點

王王稽核時必須檢查：

1. 是否符合董事長任務目標。
2. 是否只處理本次任務。
3. 是否碰到禁改清單。
4. 是否修改無關檔案。
5. 是否新增未要求功能。
6. 是否重構或整理格式。
7. 是否破壞手機版或電腦版。
8. 是否破壞已驗證功能。
9. 是否有測試與檢查結果。
10. 是否有風險需要回報。

## 7. 回報規則

完成任務後，回報必須包含：

1. 做了什麼。
2. 修改了哪些檔案。
3. 是否只修改允許檔案。
4. 是否新增功能、套件、API、資料檔或頁面。
5. 測試或檢查結果。
6. 風險或需要董事長確認的事項。
7. PR 連結。
8. 是否尚未合併。

## 8. 最終原則

王王團隊的工作目標不是把事情做大，而是把事情做準。

少改、準改、可檢查、可回退。
